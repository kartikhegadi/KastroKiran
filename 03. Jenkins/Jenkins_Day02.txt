Module 6: Jenkins_Day02
=====================
=> Throttle Builds
Team - a fresher has joined (intern) - intern has to build a job - continuously click on "build now" - load on server will get increased
Limit/restrict the number of builds to happen in a specific time period - Throttle Build

1hr ---- max 5 builds	- 60/5 = every 12 minutes you can build (1-12, 13-24, 25-....)
1hr ---- max 10 builds

=> Build Triggers - Build Periodically
Periodically - build the job at regular intervals of time
We can schedule a job - every minute, every hour, at a specific time on a specific date...
Cron Expression - * * * * * 

* * * * * ----> Every minute build the job
H * * * * ----> Every hour build the job
H/2 * * * * ----> Every 30 minutes build the job

Limitation: Build periodically doesn't bother about code changes in the GitHub repo. It will only bother about the schedule

=> Build Triggers - Poll SCM
Cron Expression - * * * * *
Poll SCM will look after the changes in the GitHub repo, and then triggers the job at the scheduled time.

Limitation: Here wait time is there

=> Build Triggers - GitHub Hook Trigger
The build will get triggered immediately after committing the code in the GitHub repo.

=> Build Triggers - Trigger Builds Remotely
Team Lead - only i can build a job
Team Members - create and configure a job

TL - went on leave

We can trigger the job by executing a Remote URL

JENKINS_URL/job/trigger-job/build?token=TOKEN_NAME

    <Valve className="org.apache.catalina.valves.RemoteAddrValve"

  <role rolename="admin-gui"/>
